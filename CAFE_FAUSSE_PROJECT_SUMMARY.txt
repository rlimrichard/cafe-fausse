Café Fausse - Project Summary
Updated: July 21, 2026

Site and deployment
-------------------
- Public site: https://ecbproj1.elcaro.io/
- Source repository: https://github.com/rlimrichard/cafe-fausse
- Production host: OCI Linux instance, accessed as the `opc` user using the
  project SSH key.
- Production application directory: /opt/cafe_fausse
- Deployment command on the host: sudo /usr/local/sbin/deploy-cafe-fausse
- HTTPS is enabled and the site is served through Nginx and Gunicorn.
- The deployment process pulls the main branch from GitHub, installs/builds
  dependencies, and restarts the application.

Email configuration and behavior
--------------------------------
- Live email is sent through the Café Fausse Gmail account using a Gmail App
  Password stored only in the protected server .env file. The credential is
  not committed to Git.
- Newsletter signups receive a welcome email describing chef specials and
  seasonal fare. Duplicate signups do not create duplicate records.
- Reservation acceptance and denial send notification emails.
- New reservation submissions now also send an immediate "request received"
  email; the reservation remains saved even if an email provider has a
  temporary delivery problem.
- Outgoing messages include both plain-text and HTML versions for better mail
  client compatibility.
- A test acceptance email to Hotmail was accepted by Gmail but went to the
  recipient's Junk folder. Marking Café Fausse mail as "Not junk" and adding
  faussecafe@gmail.com as a contact should improve inbox placement.

Recent reservation fix
----------------------
- A repeat booking could previously return a generic error because the
  customers table requires a unique email address.
- The reservation API now reuses an existing customer record (including a
  newsletter subscriber) and creates the new reservation normally.

Operational notes
-----------------
- Do not store Gmail App Passwords or private SSH keys in the repository.
- Keep the local `inspect-production-host.ps1` file untracked unless it is
  intentionally reviewed and added later.
- Current production changes are deployed from the GitHub main branch.
